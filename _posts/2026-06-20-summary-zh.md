## [探索屏幕无法显示的颜色](https://moultano.wordpress.com/2026/06/19/where-to-find-the-colors-your-screen-cant-show-you/) ⭐️ 8.0/10

一篇文章解释了屏幕为何无法显示某些饱和的蓝绿色和红色，指出了 sRGB 色域的限制，并列举了现实中的例子，如群青蓝和绿色交通灯。 这很重要，因为它揭示了人类视觉与显示技术之间的根本差距，影响了艺术家、摄影师以及任何依赖色彩准确性的人。理解这些限制可以推动采用更宽的色域标准，如 DCI-P3 或 Rec.2020。 CIE 1931 色度图显示，sRGB 仅覆盖了可见颜色的一小部分，尤其是在饱和蓝绿色和红色的再现上表现不佳。文章指出，即使是现代显示器也无法再现其三角形色域之外的颜色，并且有些颜色仅用三种原色在物理上就不可能再现。

hackernews · moultano · 6月20日 03:36 · [社区讨论](https://news.ycombinator.com/item?id=48606140)

**背景**: 像 sRGB 这样的色彩空间定义了设备可以再现的颜色子集。人眼有三种对不同波长敏感的视锥细胞，但显示器使用三种原色（红、绿、蓝）来创建一个三角形色域。许多饱和颜色，尤其是在蓝绿色和红紫色区域，落在这个三角形之外，无法准确显示。

## [ATProto 没有实例：Dan Abramov 详解](https://overreacted.io/there-are-no-instances-in-atproto/) ⭐️ 8.0/10

Dan Abramov 发表文章澄清，Bluesky 背后的 ATProto 协议没有像 Mastodon 的 ActivityPub 那样的“实例”概念，而是将托管（PDS）、聚合（Relay）和应用逻辑（AppView）分离为独立的服务。 这一澄清纠正了认为 ATProto 只是另一种带有实例的联邦协议的常见误解，帮助开发者和用户理解其独特的架构，以及为什么它相比 Mastodon 具有不同的扩展性和灵活性。 在 ATProto 中，PDS（个人数据服务器）托管用户数据，Relay 索引和分发网络中的数据，AppView 提供应用界面（例如 Bluesky 的信息流）。用户可以自由切换 PDS 提供商，而不会丢失身份或数据。

hackernews · Lobsters · 6月19日 15:10 · [社区讨论](https://news.ycombinator.com/item?id=48599515)

**背景**: Mastodon 使用 ActivityPub，每个服务器（实例）是一个自包含的社区，拥有自己的审核、存储和应用逻辑。ATProto 将这些角色解耦，旨在实现更模块化和可扩展的系统。“实例”一词是 Mastodon 思维下的概念，不适用于 ATProto。

## [Project Valhalla 历经十年终入 JDK 28](https://www.jvm-weekly.com/p/project-valhalla-explained-how-a) ⭐️ 8.0/10

Project Valhalla 的值类型和堆扁平化功能终于集成到 JDK 28 中，使得 JVM 可以在内存中密集存储对象，无需每个元素的头部或指针。 这是 JVM 的一次重大演进，显著提升了数据密集型应用的内存效率和性能，使 Java 在保持安全性的同时更接近 C 或 Rust 等语言的性能。 值类型允许用户定义具有类似基本类型性能的类型；扁平化消除了数组和字段的对象头与间接引用。但扁平化仅限于表示大小不超过 64 位的对象。

hackernews · Lobsters · 6月19日 06:35 · [社区讨论](https://news.ycombinator.com/item?id=48595511)

**背景**: 传统 Java 对象带有头部和指针等开销，导致内存膨胀和缓存未命中。Project Valhalla 始于 2014 年，旨在引入行为类似基本类型但由用户定义的值类型，在不牺牲抽象的前提下实现密集内存布局。
