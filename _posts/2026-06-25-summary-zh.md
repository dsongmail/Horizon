## [OpenAI 携手博通发布首款自研 AI 芯片 'Jalapeno'](https://techcrunch.com/2026/06/24/openai-unveils-its-first-custom-chip-built-by-broadcom/) ⭐️ 9.0/10

OpenAI 与博通共同发布了 Jalapeño，这是 OpenAI 首款自研 AI 推理芯片，由台积电制造。OpenAI 声称该芯片从设计到量产仅用九个月，并借助其自身 AI 模型加速了开发过程。 这标志着 OpenAI 的重大战略转变，减少了对英伟达 GPU 的依赖，并加入了与谷歌 TPU 及其他 ASIC 的定制 AI 芯片竞争。该芯片针对推理进行了优化，而推理正成为 AI 公司的主要利润中心。 Jalapeño 是一款光罩尺寸 ASIC，采用大型计算小芯片和 HBM 内存，以平衡高吞吐量和低延迟，适用于推理和智能体工作负载。OpenAI 表示，其架构基于对 LLM 行为的理解而设计，以解决数据搬运和内存平衡等瓶颈。

hackernews · jamdesk · 6月24日 17:47 · [社区讨论](https://news.ycombinator.com/item?id=48663324)

**背景**: AI 芯片是专门用于加速 AI 工作负载的处理器。训练芯片（如英伟达 GPU）用于构建模型，而推理芯片则运行训练好的模型以生成响应。随着 AI 从训练转向持续推理，OpenAI 等公司正在开发定制推理 ASIC 以提高效率并降低成本。

## [NVIDIA TensorRT-LLM v1.3.0rc19 新增 Wan2.2、MoE 后端和异步 Ulysses 流水线](https://github.com/NVIDIA/TensorRT-LLM/releases/tag/v1.3.0rc19) ⭐️ 8.0/10

NVIDIA 发布了 TensorRT-LLM v1.3.0rc19，新增了对 Wan2.2-T2V 量化检查点、T5、BART 和 MiniMax-M3 模型的支持，以及新的异步 Ulysses 流水线、用于 Nemotron-H 的 TRTLLM MoE 后端和用于监控的 Prometheus 指标。 此版本显著扩展了 TensorRT-LLM 的模型生态系统和优化能力，为视频生成（Wan2.2）、编码器-解码器模型（T5、BART）和 MoE 架构提供了高效推理支持，这对 NVIDIA GPU 上的生产级 AI 工作负载至关重要。 值得注意的功能包括用于重叠计算和 all-to-all 通信的异步 Ulysses 流水线、利用融合了 expand+quant 的 Triton 内核的 DeepGemmFusedMoE 新 MoE 后端，以及用于提示缓存、推测解码和批次占用的 Prometheus 指标。已知问题包括在 GB200 上 Llama 3.1 8B FP8 在自动调优预热期间可能挂起。

github · mikeiovine · 6月23日 16:49

**背景**: TensorRT-LLM 是 NVIDIA 的一个开源库，用于在 NVIDIA GPU 上优化和部署大型语言模型（LLM）和视觉生成模型。它提供 Python API 来构建 TensorRT 引擎，包含量化、动态批处理和多种并行策略等最先进的优化。异步 Ulysses 流水线是一种通信优化，它将每个 rank 的投影计算与跨 rank 的 all-to-all 传输重叠，从而提高使用 Ulysses 风格序列并行性的模型的吞吐量。

Anthropic 指控阿里巴巴通过模型蒸馏技术非法提取其 Claude AI 模型的能力，引发了关于 AI 知识产权和伦理的辩论。 这一指控凸显了围绕 AI 模型蒸馏日益紧张的局势，即公司利用专有模型的输出来训练自己的模型，引发了关于 AI 行业知识产权和合理使用的问题。 模型蒸馏涉及使用大型“教师”模型来训练较小的“学生”模型，可以通过黑盒查询或更有针对性的微调来实现。Anthropic 的指控特别针对后一种方法，该方法在行业中普遍使用。

## [LuaJIT 3.0 提议语法扩展，引发讨论](https://github.com/LuaJIT/LuaJIT/issues/1475) ⭐️ 8.0/10

LuaJIT 3.0 提议语法扩展，包括类 C 运算符（如 && 替代 and）和三目表达式（x ? y : z），如 GitHub 问题所述。这标志着可能从 Lua 传统语法转向其他语言更熟悉的约定。 这些变化可能影响游戏、嵌入式系统和高性能计算领域的数百万 LuaJIT 用户，可能提高新手的易用性，但存在分裂 Lua 生态系统的风险。这场辩论反映了语言设计中创新与兼容性之间的更广泛张力。 该提案包括逻辑运算符（&&、||、!）和三目运算符（x ? y : z）的替代语法，同时保留现有 Lua 形式。讨论还提到 Luau 的 if-then-else 表达式作为三目逻辑的侵入性较小的替代方案。

hackernews · phreddypharkus · 6月25日 00:41 · [社区讨论](https://news.ycombinator.com/item?id=48667336)

**背景**: LuaJIT 是 Lua 的即时（JIT）编译器，以其高性能著称，广泛应用于许多应用程序。Lua 本身是一种轻量级脚本语言，语法极简。该提案旨在现代化 LuaJIT 的语法，以吸引来自类 C 语言的开发者，但批评者认为这损害了 Lua 的特性，并可能造成兼容性问题。
